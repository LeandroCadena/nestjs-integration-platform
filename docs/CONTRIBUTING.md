# Contributing

## Development Workflow

1. Update documentation.
2. Implement the feature.
3. Run:

```bash
npm run format
npm run lint
npm run build
```

4. Test manually.
5. Commit using Conventional Commits.

---

## Commit Convention

- feat
- fix
- docs
- refactor
- test
- chore
- perf

---

## Engineering Principles

- Production First
- Business First
- Architecture First
- No TODOs
- Strict TypeScript
- Documentation before implementation
